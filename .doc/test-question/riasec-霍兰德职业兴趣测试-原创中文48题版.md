# Holland RIASEC 职业兴趣测试｜题目、评分规则、结果内容文档

> 设计定位：基于 Holland RIASEC 职业兴趣模型与 O*NET Interest Profiler、openpsychometrics 等公开资料，整理为适合商业产品落地的原创中文 48 题版。
>
> 本文档只保留后续页面设计与测试引擎真正需要的三部分：题目、评分规则、结果内容。研究来源放在末尾附录。
>
> 商业提醒：若直接复用官方原题文本，请按来源要求做授权与署名；当前更建议使用“结构对齐 + 中文原创改写版”。

## 1. 题目

### 1.1 作答形式

- 适合形态：`5 点量表单选`
- 题干建议：以“我愿不愿意做这类活动 / 我会不会喜欢这种任务”来发问
- 推荐选项文案：
  - `完全不想做`
  - `不太想做`
  - `一般`
  - `有点想做`
  - `非常想做`
- 分值映射：`1 / 2 / 3 / 4 / 5`
- 交互建议：
  - 单题大卡片 + 固定底部选项条
  - 不展示实时结果
  - 只展示进度，不展示维度名称
  - 六维题目全程混排，避免用户明显感知套路
  - 48 题版移动端作答时长建议控制在 `6 - 8 分钟`

### 1.2 题量建议

- `18 - 24 题`：适合免费引流版，完成率高，但结果内容会偏薄
- `36 题`：适合轻量版职业兴趣前测，能做出基本六维图谱
- `48 题`：最适合当前项目首发，六维均衡、完成率可控、结果页也足够丰富
- `60 题`：更接近官方完整体验，适合后期做更高客单的 Pro 版

### 1.3 六维结构

- `R`：现实型（Realistic）
  - 偏好动手、器械、户外、操作、制造、维修、具体可见的任务
- `I`：研究型（Investigative）
  - 偏好分析、推理、实验、理解原理、解决复杂问题
- `A`：艺术型（Artistic）
  - 偏好表达、创意、审美、构思、写作、视觉与内容创作
- `S`：社会型（Social）
  - 偏好帮助、支持、讲解、陪伴、教育、咨询、促进他人成长
- `E`：企业型（Enterprising）
  - 偏好影响、说服、推动、组织资源、争取机会、达成结果
- `C`：常规型（Conventional）
  - 偏好秩序、流程、整理、核对、记录、执行标准和稳定系统
- RIASEC 的结果应呈现为`六维兴趣图谱 + 前三码`，不建议硬包装成单一“人格类型”
- 对外表达应始终强调：`这是职业兴趣偏好，不等于能力高低，也不等于职业命运判断`

41. 我会被“把一个主意做成结果”这种过程激起斗志。
42. 我偏好清楚的计划、明确的标准和可追踪的进度。
43. 我愿意在真实环境里动手解决一个具体问题。
44. 我对观察行为、分析现象和理解人或世界的规律有兴趣。
45. 我希望自己的工作里保留一定的表达空间和个人风格。

## 2. 评分规则

### 2.1 六维归类

- `R`：1、7、13、19、25、31、37、43
- `I`：2、8、14、20、26、32、38、44
- `A`：3、9、15、21、27、33、39、45
- `S`：4、10、16、22、28、34、40、46
- `E`：5、11、17、23、29、35、41、47
- `C`：6、12、18、24、30、36、42、48

### 2.2 计分方式

- 每题按 `1 - 5` 计分
- RIASEC 活动偏好题更适合`不做反向题`
- 每个维度 `8 题`，维度原始分范围为 `8 - 40`
- 维度平均分范围为 `1.0 - 5.0`
- 建议同时保留：
  - `raw`：原始总分
  - `avg`：平均分
  - `display`：前端展示用百分制或百分比映射
  - `band`：低 / 中 / 高分区间

### 2.3 前三码生成规则

- 将六个维度按 `raw` 从高到低排序
- 取前三个维度字母，组成职业兴趣前三码
- 例如：
  - `IAS` = 研究型 + 艺术型 + 社会型
  - `SEC` = 社会型 + 企业型 + 常规型
  - `RIC` = 现实型 + 研究型 + 常规型
- 若第二名与第三名同分，结果页应显示“并列倾向”，避免制造过度精确感
- 若多个维度分差极小，建议在结果页用“你当前更偏向于……”而不是“你就是……”

### 2.4 分段建议

- `avg < 2.6`：`low`
- `2.6 ≤ avg < 3.6`：`mid`
- `avg ≥ 3.6`：`high`
- 这是产品展示用分段，不是官方常模百分位，不建议伪装成严肃常模解释

### 2.6 结果解释边界

- RIASEC 测的是`兴趣偏好`，不是能力水平
- 它可以帮助用户理解“我更愿意靠近什么样的任务与环境”，但不能直接推出“你一定适合某个职业”
- 结果页可写“更匹配的方向 / 环境 /角色特征”，不建议写成命运式结论

### 2.7 适合的策略落地方式

- `题目层`：通用答题引擎即可
- `评分层`：`accumulate + ranking + top3-code` 插件最合适
- `结果层`：建议使用`RIASEC 专用模板`
  - 六维图谱
  - 前三码大标题
  - 主导维度解释
  - 工作环境偏好
  - 适合方向与避坑提醒
- 结论：Holland RIASEC 非常适合放进“通用引擎 + 结果模板专用化”的框架内，不需要整套完全自定义页面逻辑

## 3. 结果内容

### 3.1 结果页信息架构建议

- `模块 1｜首屏结论`
  - 测试名
  - 用户昵称 / 日期
  - 前三码结果
  - 一句话轮廓总结
- `模块 2｜六维兴趣图谱`
  - 六维雷达图或六边形图
  - 每个维度 1 句短描述
- `模块 3｜主导维度报告`
  - 最高维度重点解释
  - 第二、第三维度补充说明
- `模块 4｜你偏好的任务与环境`
  - 更喜欢的任务类型
  - 更舒服的协作方式
  - 更适合的工作氛围
- `模块 5｜可能适配的方向`
  - 专业方向
  - 岗位方向
  - 项目角色方向
- `模块 6｜不适配提醒`
  - 容易消耗你的情境
  - 容易让你失去动力的工作方式
- `模块 7｜行动建议`
  - 下一步如何低成本验证自己的兴趣方向
- `模块 8｜长图导出内容`
  - 前三码封面
  - 六维图谱
  - 三张重点卡片

### 3.2 首屏结论生成规则

- 推荐模板：
  - `你的职业兴趣更偏向「{top1Name} + {top2Name} + {top3Name}」组合。`
  - `你更容易被 {top1Task}、{top2Task} 与 {top3Task} 这类事情激发兴趣。`
  - `这不是能力高低判断，而是一张你更愿意投入什么的兴趣地图。`
- 若 `top1` 与 `top2` 分差小于 `2 分`，可写：
  - `你在 {top1Name} 与 {top2Name} 两个方向都表现出很强兴趣。`
- 若 `top2` 与 `top3` 同分，建议写：
  - `你的第三兴趣方向存在并列倾向。`

### 3.3 前三码结果页生成逻辑

- `top1` 决定：核心驱动力与主标题
- `top2` 决定：补充任务偏好与协作方式
- `top3` 决定：环境氛围、表达风格或执行倾向
- 生成顺序建议：
  - 先讲 `top1`：你最容易被什么事吸引
  - 再讲 `top1 + top2`：你会如何投入与做事
  - 最后讲 `top1 + top2 + top3`：你更舒服的学习 / 工作环境
- 结果页不建议硬写“你最适合做 X 职业”
- 更好的表达是：
  - `更匹配的任务类型`
  - `更容易持续投入的环境`
  - `值得优先探索的方向`

### 3.4 六维长报告版文案

#### 3.4.1 现实型 R

**高区**

- 你通常更容易被“真实可操作”的事情吸引。
- 比起空谈，你更愿意动手做、亲自试、把东西真正搭出来。
- 器械、工具、空间、结构、维修、制作、户外或执行类任务更容易让你进入状态。
- 你通常更喜欢有明确对象和可见结果的工作，而不是长期停留在抽象讨论里。
- 提醒：高现实型不等于只能做体力或传统工种，它也可能体现在产品落地、实验操作、设备管理、影像器材、硬件体验等方向。

**中区**

- 你对动手类任务并不排斥，但也不一定把它当成最核心的兴趣来源。
- 在需要落地执行时，你通常能配合；但如果任务长期缺少思考、变化或意义，你也可能很快失去耐心。
- 你更适合“动手 + 明确目标”与其他兴趣维度结合的场景。

**低区**

- 你通常不会优先被维修、装配、操作设备或实体执行类任务吸引。
- 与其面对器材、工具和现场问题，你可能更愿意处理想法、人或信息。
- 这不表示你做不了，只是它未必能成为你长期有兴趣投入的方向。
- 提醒：当工作要求长期重复、机械、现场导向时，你可能更容易觉得耗能。

#### 3.4.2 研究型 I

**高区**

- 你通常会被“为什么会这样”“有没有更合理解释”这类问题吸引。
- 你更愿意观察、分析、拆解、研究，再得出判断。
- 数据、逻辑、模型、实验、系统理解或复杂问题求解对你更有吸引力。
- 你往往不满足于只会做，更希望理解背后的原理。
- 提醒：高研究型不等于一定走学术路线，它也可能体现为策略分析、用户研究、数据洞察、产品研究、内容研究等方向。

**中区**

- 你具备一定分析兴趣，但不一定希望长期待在纯研究环境里。
- 当研究能和创作、帮助他人、商业推进或实际应用结合时，你更容易持续投入。
- 你适合“理解问题 + 解决问题”的综合型角色。

**低区**

- 你通常不会被纯分析、长时间推理或持续钻研复杂问题强烈吸引。
- 与其长期研究原因，你可能更想直接行动、表达、协作或推动结果。
- 这不代表你缺少思考能力，而是它未必是你最能从中获得满足感的工作方式。
- 提醒：如果环境长期要求你独自深度分析，可能会让你觉得枯燥或迟滞。

#### 3.4.3 艺术型 A

**高区**

- 你通常会被表达、创意、审美、风格、故事感和“做出自己的版本”吸引。
- 你更在意一件事是否有表达空间，而不只是是否高效完成。
- 写作、内容、视觉、影像、音乐、策划、创意构思或品牌表达更容易激发你的兴趣。
- 你可能需要一定自由度，才能真正进入状态。
- 提醒：高艺术型不等于一定要成为传统艺术工作者，它也可能体现在内容、品牌、体验设计、创意营销、产品表达等方向。

**中区**

- 你欣赏创意和审美，但也不一定要求所有工作都完全自由发挥。
- 当表达能服务于目标、项目或具体结果时，你更容易发挥稳定。
- 你适合“创意 + 结构”平衡型场景。

**低区**

- 你通常不会把“自我表达”视为工作中最核心的满足来源。
- 比起自由发挥，你可能更喜欢明确任务、稳定路径或已知标准。
- 这不表示你没有审美，只是创作本身未必是你愿意长期投入的主轴。
- 提醒：当环境高度模糊、评价标准很主观时，你可能会更容易不适应。

#### 3.4.4 社会型 S

**高区**

- 你通常会被“帮助别人变得更好”这件事打动。
- 你更容易从解释、支持、陪伴、指导、协作和照顾他人中获得意义感。
- 教育、咨询、服务、社群、辅导、培训、沟通支持类任务更可能让你有投入感。
- 你在意人与人之间是不是被看见、被理解、被照顾到。
- 提醒：高社会型不等于只能做助人行业，它也可能体现在客户成功、用户运营、面试辅导、知识服务、社区经营等方向。

**中区**

- 你对帮助别人这件事有一定意愿，但也需要看到边界和回报。
- 当支持他人和推进目标能同时发生时，你往往更有动力。
- 你适合“人际协作 + 明确任务”的中间型位置。

**低区**

- 你通常不会把持续陪伴、答疑、照顾情绪或提供支持当作最自然的投入方向。
- 与其长期处理人际需求，你可能更愿意处理内容、系统、创意或结果本身。
- 这不等于你不善良，只是高频照顾他人未必是你最舒服的工作方式。
- 提醒：若岗位长期需要大量情绪劳动，你可能更容易被消耗。

#### 3.4.5 企业型 E

**高区**

- 你通常会被机会、结果、影响力和推进感激活。
- 你不太排斥竞争、表达、争取、提案、谈判、销售、带动他人或把事做成。
- 你更容易在目标导向强、节奏快、需要主动推进的场景中进入状态。
- 你往往愿意承担“把局面往前带”的角色。
- 提醒：高企业型不等于一定外放张扬，它也可能体现为项目 owner 意识、资源整合、商业嗅觉和推动闭环的能力。

**中区**

- 你并不抗拒表达和推动，但也未必想长期处在强竞争前线。
- 当你认同目标、看见机会、或者知道这件事值得推进时，你能展现不错的行动力。
- 你适合“有一定主导空间，但不必时时高压争抢”的环境。

**低区**

- 你通常不会优先被销售、竞争、说服、争取资源或带队冲结果这类情境吸引。
- 比起主导局面，你可能更愿意沉下心做内容、做研究、做支持或做系统。
- 这不表示你没有执行力，只是强商业推进未必是你的主要兴趣来源。
- 提醒：如果岗位持续要求你高频对外、强势推进和承担业绩压力，你可能更容易失去动力。

#### 3.4.6 常规型 C

**高区**

- 你通常对秩序、规则、结构、计划、准确性和稳定推进更敏感。
- 你会因为流程清晰、信息整齐、任务有序、系统可控而感到安心。
- 整理、归档、核对、排期、流程、执行标准、后台支持这类任务更容易让你保持稳定表现。
- 你通常不排斥细节，也更愿意把事情做完整。
- 提醒：高常规型并不意味着保守或缺乏创造力，它同样可能出现在项目管理、运营体系、财务支持、行政流程、数据清洗等方向。

**中区**

- 你对秩序有基本需求，但也能接受一定变化和弹性。
- 当任务既有清晰框架、又保留一些自主空间时，你通常会发挥得更舒服。
- 你适合“结构明确，但不完全僵化”的工作环境。

**低区**

- 你通常不会因为整理、录入、反复核对、流程执行本身而感到强烈满足。
- 比起稳定重复的系统工作，你可能更想做有变化、有表达或更有互动感的事情。
- 这不等于你做不好细节，而是它不一定能成为让你长期有兴趣的主轴。
- 提醒：若环境非常强调规则、流程、低变动和高重复，你可能更容易觉得压抑。

- 若 `top1 = S`
  - 你更容易在`解释、辅导、支持、安抚、陪练、连接他人`的片段里体现价值。

- 若 `top1 = C`：更像`秩序维护者 / 流程负责人 / 节奏控制者 / 交付保障者`

- 若 `top2 = I`：补充“你通常不只想做完，还想知道为什么这样做才对。”

#### 模块 5｜更具体的探索方向样例

> 这里不是“命定职业名单”，而是更值得优先试水的方向样例。页面上可以根据 `top1 + top2` 动态取 6 - 10 条展示。

- `R` 高时可优先探索：
  - 摄影器材 / 影像设备助理

  - 视觉设计、插画、摄影、审美策划、包装表达
  - 公众号、小红书、播客、个人栏目等内容型副业

- `S` 高时可优先探索：
  - 教育辅导、训练营助教、知识服务答疑
  - 社群运营、用户成功、咨询助理、HR 支持
  - 生涯辅导、面试陪练、留学申请协助、教练型服务

#### 模块 6｜常见误配环境提醒

- `R` 低时：长期高现场、高器械、高重复操作的工作更容易让你失去兴趣。
- `I` 低时：长期独立分析、反复查证、纯研究导向环境可能会让你觉得拖沓。
- `A` 低时：高度模糊、极度主观、长期“自由发挥”的环境可能让你不安。
- `S` 低时：高情绪劳动、持续陪伴型、强服务导向岗位可能更容易消耗你。
- `E` 低时：持续谈判、强目标压迫、频繁对外争取的环境可能会让你想退开。
- `C` 低时：高重复、重流程、低变化、强细节校验的岗位更容易让你觉得压抑。
- 结果页建议固定展示 `lowest` 维度对应的一条提醒，这会比泛泛夸赞更让用户觉得真实。

#### 模块 7｜Top1 + Lowest 的具体化生成逻辑

- 若 `top1 = I` 且 `lowest = E`
  - 可写：`你更像“先想明白再行动”的人，适合深度分析、研究支持、内容拆解类角色，但若岗位要求你持续对外推进、频繁谈判和高压成交，消耗感可能会来得更快。`
- 若 `top1 = A` 且 `lowest = C`
  - 可写：`你有明显表达欲和创作驱动力，但如果缺少最基本的时间结构与交付机制，灵感很容易停留在脑海里。你更适合“创意主导 + 轻运营支持”的环境。`
- 若 `top1 = S` 且 `lowest = R`
  - 可写：`你更容易在人与人的互动里发光，但若工作长期是冰冷设备、机械执行和低互动现场，你可能会很难获得持续投入感。`
- 若 `top1 = E` 且 `lowest = I`
  - 可写：`你擅长带动与推进，但如果岗位长期要求你一个人慢慢查证、反复推理、迟迟不落地，你可能会觉得节奏太慢。`
- 若 `top1 = C` 且 `lowest = A`
  - 可写：`你很适合稳定交付、流程维护和细节把控，但在完全没有标准答案、频繁需要天马行空表达的环境里，可能会不太舒服。`
- 若 `top1 = R` 且 `lowest = S`
  - 可写：`你更愿意处理事与物，而不是长时间承接别人的情绪与需求；岗位若要求高频安抚、长期陪伴或重服务表达，可能会加速消耗。`

#### 模块 8｜行动建议生成逻辑

- 若 `top1 = R`：建议用户优先去做一次`强实操体验`，比如器材拍摄、空间搭建、烘焙、园艺、现场执行。
- 若 `top1 = I`：建议用户优先做一次`研究型小项目`，比如专题调研、数据分析、竞品拆解、测评文章。
- 若 `top1 = A`：建议用户优先做一次`表达型输出`，比如短视频脚本、图文栏目、视觉作品、品牌表达页。
- 若 `top1 = S`：建议用户优先做一次`助人型试水`，比如答疑、带练、陪练、课程助教、社群支持。
- 若 `top1 = E`：建议用户优先做一次`推进型尝试`，比如提案、拉合作、做转化页、组织小活动。
- 若 `top1 = C`：建议用户优先做一次`流程优化型任务`，比如整理资料库、排期系统、课程运营表、SOP 梳理。
- 行动建议最好落到`一周内可以开始的小尝试`，而不是抽象鸡汤。

### 3.8 优先制作的 12 组三码结果草稿（产品模板）

> 说明：以下 12 组不是官方划分的“固定类型”，也不是统计意义上的“全国高频结果”。
>
> 它们是更适合当前产品先做出来的 `首批结果模板`：覆盖面广、页面好做、内容也更容易让用户觉得“具体、像在说我”。
>
> 页面上建议把这部分作为 `前三码专属补充报告`，放在通用六维结果之后。

#### 1. `IAS`｜研究 × 艺术 × 社会

- **核心轮廓**：你通常既想把事情想明白，也想把它讲明白，还希望这些表达最终能真正帮到别人。
- **更容易发光的工作片段**：拆解复杂问题、把干货转成容易理解的内容、把抽象知识做成有吸引力的表达。
- **团队里的自然位置**：研究型内容策划、知识产品设计者、用户洞察转译者、课程内容搭建者。
- **更值得优先试水的方向**：用户研究、知识型内容、小红书科普账号、课程策划、测评内容设计、教育内容产品。
- **误配提醒**：如果岗位只有纯分析却没有表达出口，或者只有高频表达却没有深度内容支撑，你都可能做一段时间就开始厌倦。

#### 2. `IAC`｜研究 × 艺术 × 常规

- **核心轮廓**：你不仅想理解问题，也希望把复杂信息整理得更清楚、更美观、更可被稳定复用。
- **更容易发光的工作片段**：做结构化资料、整理知识体系、把复杂内容做成图表、页面、手册、教程或标准模板。
- **团队里的自然位置**：信息架构设计者、课程资料整理者、研究报告编辑者、内容系统搭建者。
- **更值得优先试水的方向**：信息设计、内容编辑、课程设计、知识库搭建、研究助理、题库与测评产品内容整理。
- **误配提醒**：如果环境长期只有创意发散、没有结构沉淀，或者只有流程搬运、没有思考空间，你都会觉得缺点什么。

#### 3. `ISE`｜研究 × 社会 × 企业

- **核心轮廓**：你更像那种“先看懂问题，再去推动解决”的人，适合连接洞察、沟通与结果。
- **更容易发光的工作片段**：访谈需求、诊断问题、做解决方案、向别人解释逻辑并推进落地。
- **团队里的自然位置**：咨询型顾问、解决方案支持者、培训项目策划、研究转商业的桥梁角色。
- **更值得优先试水的方向**：咨询助理、用户研究转产品、课程顾问、方案策划、知识服务转化、商业分析支持。
- **误配提醒**：如果岗位要求你只做深度研究却完全不和人接触，或者只追销量却不允许你理解问题本身，你都可能失去耐心。

#### 4. `SAE`｜社会 × 艺术 × 企业

- **核心轮廓**：你往往擅长让内容更有吸引力，也更擅长让人愿意参与、愿意被打动、愿意跟着一起行动。
- **更容易发光的工作片段**：组织活动、讲解内容、营造氛围、做互动表达、把一个主题讲得既好懂又有感染力。
- **团队里的自然位置**：社群主理人、活动策划、内容型讲述者、训练营运营、品牌互动角色。
- **更值得优先试水的方向**：社群运营、课程主理、活动策划、品牌内容、直播内容、教育型自媒体、成长产品运营。
- **误配提醒**：如果岗位几乎没有人与人互动，只有后台流程和重复执行，你通常很难长期保持热情。

#### 5. `SEC`｜社会 × 企业 × 常规

- **核心轮廓**：你既在意人，也在意推进，还能接受结构化执行，是非常适合“把服务做稳、把项目带顺”的组合。
- **更容易发光的工作片段**：对接用户、推进流程、协调资源、维护节奏、把混乱事项梳理成可执行方案。
- **团队里的自然位置**：项目协调者、客户成功、班主任型运营、课程运营、HR / 行政支持中的对人岗位。
- **更值得优先试水的方向**：用户成功、社群运营、教务、课程运营、项目协调、招聘运营、门店管理。
- **误配提醒**：如果岗位既没有人际互动，也没有推进空间，只剩机械重复录入，你可能会觉得价值感很低。

#### 6. `SIA`｜社会 × 研究 × 艺术

- **核心轮廓**：你很适合做“理解人、观察人、再把这种理解转成表达或帮助”的事情。
- **更容易发光的工作片段**：倾听、访谈、观察情绪与需求、提炼洞察、把复杂感受翻译成清晰内容。
- **团队里的自然位置**：用户访谈者、咨询支持者、教育内容设计者、心理 / 生涯内容创作者。
- **更值得优先试水的方向**：用户研究、心理内容账号、访谈型内容、生涯辅导支持、教育内容、成长型服务设计。
- **误配提醒**：如果岗位强竞争、强成交、强对外施压，而没有耐心理解人的空间，你会很快感到别扭。

#### 7. `AEC`｜艺术 × 企业 × 常规

- **核心轮廓**：你不只是想做出好看的表达，还希望它能带来结果，并且能稳定执行下去。
- **更容易发光的工作片段**：做营销内容、包装卖点、搭视觉表达、改页面、做活动方案并跟进落地。
- **团队里的自然位置**：品牌内容策划、营销创意执行、内容商业化负责人、增长内容角色。
- **更值得优先试水的方向**：品牌内容、电商内容、营销策划、短视频商业化、活动策划、页面表达优化。
- **误配提醒**：如果环境只有纯创意没有结果反馈，或者只有流程执行没有表达空间，你都不容易真正满意。

#### 8. `AIC`｜艺术 × 研究 × 常规

- **核心轮廓**：你适合做“有表达感、又有结构和逻辑”的内容，不只是灵感型创作，而是能沉淀成方法的创作。
- **更容易发光的工作片段**：搭内容框架、做栏目体系、整理知识表达、打磨页面结构与文案逻辑。
- **团队里的自然位置**：内容系统设计者、UX 文案、课程内容编辑、结构化创作型角色。
- **更值得优先试水的方向**：内容策划、UX Writing、课程内容、知识图文、自媒体栏目设计、题库内容整理。
- **误配提醒**：如果岗位要求你持续高频粗放输出、完全不允许深思熟虑和打磨结构，你可能会越来越痛苦。

#### 9. `ERC`｜企业 × 现实 × 常规

- **核心轮廓**：你更像那种“把事情真推进、真落地、真交付”的人，比起想法，你更在意结果有没有发生。
- **更容易发光的工作片段**：拉资源、控现场、盯进度、解决执行阻塞、把项目从纸面推到现实。
- **团队里的自然位置**：活动执行负责人、门店运营、商务执行、项目落地 owner、现场统筹。
- **更值得优先试水的方向**：活动执行、门店管理、项目实施、会务统筹、供应链执行、运营落地岗位。
- **误配提醒**：如果岗位充满概念讨论、迟迟不落地、责任边界模糊，你通常会比别人更早失去耐心。

#### 10. `RIC`｜现实 × 研究 × 常规

- **核心轮廓**：你适合那种既要动手、又要判断、还要按标准稳定完成的任务。
- **更容易发光的工作片段**：测试、校验、调试、做样本、看数据、排故障、按标准把事情做准。
- **团队里的自然位置**：实验 / 测试支持、质量控制、产品打样协助、数据采集与验证角色。
- **更值得优先试水的方向**：实验室助理、测试支持、品控、硬件体验、数据采集、流程质检、教培测评执行。
- **误配提醒**：如果环境长期只讲抽象愿景、不给具体标准，也不给实际操作空间，你会很难找到投入感。

#### 11. `RSE`｜现实 × 社会 × 企业

- **核心轮廓**：你往往适合在“真实场景 + 需要带人 + 需要推进”的位置上发挥，不喜欢空转，更喜欢真接触、真协调、真解决。
- **更容易发光的工作片段**：带活动现场、照顾参与者体验、临场协调、边做边调整、把一群人带进同一节奏。
- **团队里的自然位置**：现场负责人、门店主理、训练营执行主理、体验运营、带班型角色。
- **更值得优先试水的方向**：活动现场、门店主理、训练营运营、宠物 / 运动 / 手作体验项目、线下服务主理。
- **误配提醒**：如果岗位长期只有文档、表格和幕后流程，没有真实场景和人际反馈，你可能很难保持兴趣。

#### 12. `CIE`｜常规 × 研究 × 企业

- **核心轮廓**：你更像“把复杂事情理顺，再推进它跑起来”的人，兼具结构感、分析力和一定目标意识。
- **更容易发光的工作片段**：梳理流程、看数据、找问题、提优化方案、推动协作链条变顺。
- **团队里的自然位置**：运营策略支持、项目管理、数据运营、商业分析、流程优化角色。
- **更值得优先试水的方向**：数据运营、项目管理助理、策略运营、业务分析、SOP 优化、供应链协同、教务管理。
- **误配提醒**：如果岗位极度随性、天天临时改、没有标准也没有复盘机制，你很容易感觉混乱又低效。

#### 使用建议

- 页面层建议先做这 `12 组`，足以支撑第一版产品上线。
- 兜底策略建议：
  - 若命中这 12 组之一，优先渲染专属结果块。
  - 若未命中，则使用 `top1 + top2 + top3 + lowest` 的通用拼装结果。
- 后续若要继续扩容，不必一次性补满 216 组，优先补：
  - 出现频率高的组合
  - 商业化表达强的组合
  - 页面视觉好做、内容差异大的组合

### 3.9 首批 6 组三码深度结果稿（可直接映射结果页）

> 说明：这一节不是新的理论层，而是为了让后续结果页真正“能上线、像产品”。
>
> 每组都按页面可复用的字段来写：`首屏标题`、`结果摘要`、`工作片段`、`团队位置`、`试水方向`、`误配环境`、`长图摘句`。
>
> 前端实现时，可以把它们直接转成结构化 schema。

#### 1. `IAS` 深度结果稿

- **首屏标题**：`洞察表达者`
- **结果摘要**：
  - 你通常不满足于只知道结论，你更想把一件事看懂、拆开，再用更容易被接受的方式表达出来。
  - 你既需要思考的深度，也需要表达的出口；如果一份工作只能研究不能输出，或者只能输出不能深入，你都可能做不久。
  - 你更容易在“理解复杂问题 → 提炼重点 → 讲给别人听 / 做给别人看”这条链路里进入状态。
- **你会喜欢的工作片段**：
  - 把复杂主题写成清楚的图文、脚本、课程或测评结果
  - 做访谈、读资料、拆案例，然后提炼出有价值的观点
  - 把原本枯燥的知识，转成更有吸引力、更容易传播的表达
  - 给一个主题搭结构：先讲什么、后讲什么、什么该重点强调
- **你在团队里的自然位置**：
  - 研究型内容策划
  - 用户洞察转译者
  - 知识产品 / 课程内容设计者
  - 需要“先想明白，再讲明白”的中间角色
- **更值得优先试水的方向**：
  - 用户研究 + 内容输出
  - 科普型账号 / 知识型自媒体
  - 课程策划 / 测评内容设计
  - 访谈栏目 / 研究型图文 / 长内容专题
  - 教育内容产品 / 知识服务产品
- **误配环境**：
  - 只有纯传播 KPI、没有内容深度的岗位
  - 长期只做研究支持、完全没有表达出口的岗位
  - 需要高频机械复制、却不允许你整理结构和观点的环境
- **长图摘句**：
  - `你最容易发光的时刻，是把复杂的东西讲清楚。`
  - `你不是只想输出观点，你更想输出“有洞察的表达”。`

#### 2. `IAC` 深度结果稿

- **首屏标题**：`结构化创作人`
- **结果摘要**：
  - 你更容易被“复杂内容如何变得更清楚、更好看、更可复用”这件事吸引。
  - 你不是纯感性的创作者，也不是只会归档的人；你更像那种能把思考、表达和结构整合起来的人。
  - 当一份工作既有逻辑，又允许你打磨呈现方式时，你通常会做得很稳。
- **你会喜欢的工作片段**：
  - 给知识、课程、信息或页面搭结构
  - 把零散资料整理成手册、模板、页面、指南或知识库
  - 调整内容顺序、标题、层级，让它更顺、更清楚、更耐用
  - 在表达里保留审美感，但又不牺牲交付与逻辑
- **你在团队里的自然位置**：
  - 信息架构设计者
  - 课程资料整理者
  - 内容系统搭建者
  - 结构型编辑 / UX 文案 / 知识整理角色
- **更值得优先试水的方向**：
  - 课程内容编辑
  - 知识库 / 帮助手册搭建
  - 信息设计 / 页面信息结构整理
  - 题库、测评、学习产品的内容整理与表达优化
  - 有逻辑要求的内容型岗位
- **误配环境**：
  - 只有发散创意、完全没有结构沉淀的环境
  - 只有重复录入、没有思考空间的低价值整理工作
  - 内容节奏极乱、天天推翻重来、没有任何方法感的团队
- **长图摘句**：
  - `你擅长的，不只是创作，而是把内容做得更清楚、更耐看、更能复用。`
  - `你会因为“混乱被整理清楚”而获得成就感。`

#### 3. `ISE` 深度结果稿

- **首屏标题**：`问题解决顾问型`
- **结果摘要**：
  - 你往往更像那种“先看懂问题，再去推动解决”的人。
  - 你对人有感知，对问题有分析欲，对结果也有一定推动力，因此很适合站在“理解需求”和“给出方案”之间。
  - 你不一定喜欢最前线的强销售角色，但你通常适合做有判断、有解释、有推动的解决方案型工作。
- **你会喜欢的工作片段**：
  - 和人聊清楚需求、困扰和真实卡点
  - 把零散问题整理成可执行的思路和建议
  - 做调研、诊断、分析，再把结果讲给需要的人听
  - 推动一个原本模糊的问题进入“有人能开始做”的状态
- **你在团队里的自然位置**：
  - 咨询助理 / 解决方案支持
  - 培训项目策划
  - 研究转商业的桥梁角色
  - 用户问题诊断与方案转译者
- **更值得优先试水的方向**：
  - 咨询助理 / 顾问支持
  - 用户研究转产品 / 课程顾问 / 内容顾问
  - 商业分析支持 / 培训方案策划
  - 需要“先理解后推动”的项目型岗位
- **误配环境**：
  - 只做深度研究、完全不接触真实人和需求的岗位
  - 只追成交结果、却不允许你理解问题结构的岗位
  - 长期情绪劳动很重、但无法形成方案沉淀的服务工作
- **长图摘句**：
  - `你更适合解决问题，而不是只围着问题打转。`
  - `你能把理解、分析和推动，接成一条真正有结果的链路。`

#### 4. `SAE` 深度结果稿

- **首屏标题**：`感染式带动者`
- **结果摘要**：
  - 你通常擅长把一个主题讲得更有吸引力，也更容易把人带进某种氛围和行动状态里。
  - 你往往不是靠冷冰冰的说服，而是靠内容感、互动感和情绪连接，让别人愿意参与、愿意靠近、愿意被触发。
  - 如果一份工作既能接触人，又允许你做表达和组织，你通常会越做越有劲。
- **你会喜欢的工作片段**：
  - 组织活动、控现场氛围、把人带入同一节奏
  - 把主题讲得好懂、有感染力、有参与感
  - 做训练营、社群、品牌活动、直播或互动内容
  - 让原本普通的信息，变成让人想参与的一次体验
- **你在团队里的自然位置**：
  - 社群主理人
  - 活动策划 / 互动内容角色
  - 训练营运营 / 品牌互动角色
  - 教育型内容表达者
- **更值得优先试水的方向**：
  - 社群运营 / 训练营 / 直播内容
  - 活动策划 / 品牌互动 / 教育型账号
  - 课程主理 / 成长型社媒 / 用户互动产品
  - 需要“讲给人听、带着人做”的角色
- **误配环境**：
  - 长期后台、低互动、纯表格和重复流程的岗位
  - 表达被压缩到最低、只能机械执行的环境
  - 高冷、弱互动、没有情绪反馈的团队氛围
- **长图摘句**：
  - `你擅长的不是安静输出，而是把人真正带进来。`
  - `当一个主题需要被讲活、被带动、被点燃时，你更容易发光。`

#### 5. `SEC` 深度结果稿

- **首屏标题**：`服务推进协调者`
- **结果摘要**：
  - 你既在意人，也在意事情能不能真的被推进和落实。
  - 你不是那种只会安抚别人的人，你通常还希望流程顺、节奏稳、结果别掉链子。
  - 这让你很适合做“对人 + 对项目”都需要兼顾的岗位：既能协调，也能执行，还能让人感觉被接住。
- **你会喜欢的工作片段**：
  - 对接用户、跟进事项、处理反馈、维护节奏
  - 把混乱事情梳理成大家都能配合的推进方案
  - 在服务过程中发现问题并顺手补上流程漏洞
  - 让一个班级、社群、项目或团队运转得更稳
- **你在团队里的自然位置**：
  - 项目协调者
  - 用户成功 / 客户成功
  - 课程运营 / 教务 / 班主任型角色
  - 对人服务中的推进型岗位
- **更值得优先试水的方向**：
  - 课程运营 / 教务 / 社群管理
  - 用户成功 / 项目协调 / 招聘运营
  - 训练营、服务型产品、知识服务的执行与维护
  - 需要稳定对接与持续跟进的岗位
- **误配环境**：
  - 没有人际互动，也没有推进空间，只剩机械重复执行
  - 目标和流程都混乱，却又没人真正负责收口的环境
  - 只要求“情绪价值”，却不给你结构和权限的岗位
- **长图摘句**：
  - `你适合把人照顾好，也把事推进顺。`
  - `你真正擅长的，是让一段服务体验变得稳定、顺畅、可靠。`

#### 6. `SIA` 深度结果稿

- **首屏标题**：`理解型助人表达者`
- **结果摘要**：
  - 你更容易被“理解一个人真实在经历什么，并把这种理解转成有帮助的内容或陪伴”所打动。
  - 你通常既有对人的感受力，也有观察和提炼的能力，还希望表达不只是好看，而是真的能帮到人。
  - 当一份工作允许你倾听、理解、整理和表达同时发生时，你更可能长期投入。
- **你会喜欢的工作片段**：
  - 做访谈、倾听故事、观察需求和情绪变化
  - 把一个人混乱的状态整理成可被理解的语言
  - 做成长内容、陪伴型服务、解释型内容、访谈型内容
  - 让别人因为你的表达而更理解自己、更敢往前一步
- **你在团队里的自然位置**：
  - 用户访谈者
  - 成长型内容创作者
  - 教育内容设计者
  - 咨询支持 / 生涯辅导支持 / 心理内容角色
- **更值得优先试水的方向**：
  - 访谈型内容 / 心理成长内容 / 生涯内容
  - 用户研究 / 陪伴型社群 / 教育内容 / 课程支持
  - 面试陪练 / 学习陪伴 / 成长型知识产品
  - 需要“理解人并把理解讲出来”的岗位
- **误配环境**：
  - 高压成交、强竞争、只认结果不认人的环境
  - 长期处理冰冷流程、完全没有人与人互动的岗位
  - 需要持续强势输出，却不给你理解和观察空间的角色
- **长图摘句**：
  - `你最自然的能力，是把理解变成帮助。`
  - `你不只是会共情，你更会把这种理解整理成别人真正用得上的东西。`

#### 前端落地建议

- 这 6 组可以先做成 `resultPresetMap`。
- 每组至少拆成以下字段：
  - `heroTitle`
  - `summaryParagraphs[]`
  - `favoriteMoments[]`
  - `teamRole[]`
  - `tryFirst[]`
  - `mismatchWarnings[]`
  - `posterQuotes[]`
- 页面上建议采用：
  - 首屏 1 句标题 + 1 段摘要
  - 中段 3 个卡片：工作片段 / 团队位置 / 试水方向
  - 尾段 1 个误配提醒 + 1 个行动建议
- 这会比只展示六维图谱更像一个真正值钱的测试结果页。
