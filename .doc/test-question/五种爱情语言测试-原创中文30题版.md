- 更新时间：2026-03-14
- 文档用途：为 `SoulTest` 的测试页、结果页、长图页设计提供可直接落地的内容资产。
- 当前范围：只保留 `题目`、`评分规则`、`结果内容` 三部分；研究来源放在文末附录。
- 使用原则：本稿基于 Gary Chapman 提出的 `五种爱情语言` 概念框架整理，但不直接复用官方 30 题文案；题面为适配中文移动端体验而写的 `原创中文改编版`，可用于产品化测试设计。
- 风险提示：`The 5 Love Languages®` 为官方品牌表达。若前台用于商业售卖，建议产品名优先使用 `爱的接收偏好测试` / `亲密关系偏好测试`，副标题再写“基于五种爱情语言模型改编”，避免让用户误以为是官方授权版本。
- 表达边界：这套测试更适合作为 `关系沟通偏好` 的内容产品，不适合作为人格优劣、关系对错或心理诊断结论。

## 1. 题目

### 1.1 作答形式

- 题型：`配对二选一`
- 题量：`30 题`
- 维度：`肯定的言辞`、`精心的时刻`、`接受礼物`、`服务的行动`、`身体的接触`
- 核心交互：每题同时展示两条都“不错”的亲密关系表达方式，让用户选出 `更容易让自己感到被爱` 的那一条
- 前台提示语建议：`不是选“喜不喜欢”，而是选“哪一种更能打动你”。`
- 页面节奏建议：`一题一卡`、`选后自动进入下一题`、`顶部进度条固定`、`支持上一题返回修改`
- 视觉建议：两张等权重卡片并列；不要把某一项做成主按钮，避免引导性偏差
- 场景建议：题面尽量贴近中文恋爱语境，覆盖 `消息表达`、`见面陪伴`、`纪念日`、`日常照顾`、`身体靠近` 等高频亲密关系场景

### 1.2 五个维度的前台定义

#### 1.2.1 肯定的言辞 `words_of_affirmation`

- 你更容易被：`明确说出来的欣赏、感谢、鼓励、重视感` 打动
- 你在意的不只是“有爱”，而是 `爱有没有被认真表达出来`

#### 1.2.2 精心的时刻 `quality_time`

- 你更容易被：`被专注陪伴、被认真倾听、被优先安排时间` 打动
- 你在意的不只是“见面”，而是 `对方是否真的把注意力给了你`

#### 1.2.3 接受礼物 `receiving_gifts`

- 你更容易被：`被惦记、被准备、被用具体心意记住` 打动
- 你在意的不只是“东西”，而是 `那份被放在心上的证据`

#### 1.2.4 服务的行动 `acts_of_service`

- 你更容易被：`被照顾、被分担、被实际行动支持` 打动
- 你在意的不只是“态度”，而是 `对方是否真的愿意动手`

#### 1.2.5 身体的接触 `physical_touch`

- 你更容易被：`拥抱、牵手、依偎、靠近、安抚性的触碰` 打动
- 你在意的不只是“亲密感”，而是 `身体距离是否真的拉近`

### 1.3 选项与计分映射

- 每题两个选项分别映射到两个不同维度
- 用户每题只能选一个选项
- 被选中的那个维度 `+1`
- 30 题做完后，得到五个维度的分布结果

维度键名建议：

| 中文维度 | 字段 key | 短标签 |
| --- | --- | --- |
| 肯定的言辞 | `words_of_affirmation` | `言辞` |
| 精心的时刻 | `quality_time` | `时刻` |
| 接受礼物 | `receiving_gifts` | `礼物` |
| 服务的行动 | `acts_of_service` | `行动` |
| 身体的接触 | `physical_touch` | `接触` |

## 2. 评分规则

### 2.1 计分方式

每题只记一次分：

- 若用户选择左侧选项，则对应的维度 `+1`
- 若用户选择右侧选项，则对应的维度 `+1`
- 最终得到五个维度分数

```text
words_of_affirmation = 被选中的“肯定的言辞”题数
quality_time = 被选中的“精心的时刻”题数
receiving_gifts = 被选中的“接受礼物”题数
acts_of_service = 被选中的“服务的行动”题数
physical_touch = 被选中的“身体的接触”题数
```

### 2.2 分值范围

- 总题量：`30`
- 每个维度在整套题中共出现 `12` 次
- 因此每个维度的理论分值范围为：`0 ~ 12`
- 五维总分恒为：`30`

### 2.3 排名逻辑

- `primaryLanguage`：分数最高的维度
- `secondaryLanguage`：分数第二高的维度
- `lowestSensitivity`：分数最低的维度

更适合前台展示的判定建议：

- 若第一名与第二名分差 `>= 2`：展示为 `主导爱情语言明确`
- 若第一名与第二名分差 `<= 1`：展示为 `双主导爱情语言`
- 若五维分布相对接近，例如 `最高分 - 最低分 <= 2`：可补充标签 `整体偏均衡`

说明：

- `双主导` 和 `整体偏均衡` 都是产品展示策略，不冒充官方诊断式 cut-off
- 对这类偏好型测试，更建议用 `排序 + 差值 + 分布图`，而不是硬切“高低优劣”

### 2.4 百分比换算建议

为了结果页与长图展示更直观，可增加一个标准化百分比：

```text
languagePercent = score / 12 * 100
```

示例：

- `12 分` => `100%`
- `9 分` => `75%`
- `6 分` => `50%`

注意：

- 这里的百分比只代表 `在本套题中的相对偏好强度`
- 不代表人群百分位，也不代表某种“关系能力更高”

### 2.5 推荐结果展示规则

结果页建议固定输出以下结构：

- `主语言`：你最容易感到被爱的方式
- `次语言`：与你的主语言一起构成关系偏好的第二通道
- `五维分布图`：建议用 `雷达图` 或 `五维柱图`
- `一句总评`：用更像人话的方式总结你的关系偏好
- `主语言深读`：解释什么最能打动你、什么最容易让你失望
- `次语言补充`：解释第二需求通道
- `低敏感区提醒`：说明“分低不等于排斥，只是相对没那么敏感”
- `伴侣行动建议`：给出立刻能做的表达建议

## 3. 结果内容

### 3.1 结果页信息架构建议

#### 3.1.1 页面目标

- 第一眼就回答：`你最容易被哪一种方式打动？`
- 第二屏开始让用户感觉：`这不是一句标签，而是自己在关系里真正会在意的细节`
- 页面末尾要自然承接：`保存长图`、`分享结果`、`去看别的测试`
- 内容调性要温和、具体、可执行，不要写成高高在上的情感说教

#### 3.1.2 推荐页面结构（移动端优先）

1. `Hero 结果首屏`
2. `五维分布图`
3. `主语言深读`
4. `次语言补充`
5. `你最有感觉的表达方式`
6. `你最容易误会或失落的瞬间`
7. `伴侣可执行建议`
8. `保存长图 / 再测一套`

#### 3.1.3 Hero 首屏必备信息

- `主标题`：直接给出主语言或双主导判断
- `副标题`：一句解释你的关系感受机制
- `五维分布迷你图`
- `主语言 + 次语言标签`
- `一句短评`
- `CTA`：`保存结果`、`生成长图`、`返回首页`

Hero 文案风格示例：

- `你的被爱雷达，最先亮起的是「精心的时刻」`
- `你不是不需要爱，你需要的是被认真放在当下。`
- `你的结果是「肯定的言辞 × 服务的行动」双主导`
- `光说不做不行，只做不说也不够。你更容易被“说到 + 做到”一起打动。`

#### 3.1.4 分布图区建议

这一屏的任务不是解释全部，而是让用户先看懂结构：

- 五维分数排序
- 第一名与第二名差值
- 最低敏感区提示
- 图下同步给出文字版分数，避免图表一眼看不懂

建议可视化：

- 首选：`五维雷达图`
- 备选：`横向五维柱图`
- 移动端简化方案：`五张分值胶囊卡`

#### 3.1.5 主语言深读区建议

这一屏是“值回票价”的关键内容，建议包含：

- `你最容易有感觉的触发点`
- `你会把什么当作爱的证据`
- `你最容易因为什么而失落`
- `最适合你的关系修复方式`
- `一句伴侣可直接照着做的建议`

#### 3.1.6 次语言补充区建议

不要把结果做成“只有一个标签”。很多用户真正的体验是两条通道都很重要。

这一屏建议说明：

- 次语言在关系中如何补充主语言
- 为什么别人容易只满足你的一部分，却还是让你觉得“不够”
- 当主语言和次语言被同时满足时，用户会最有“被理解感”

#### 3.1.7 低敏感区提醒

结果页里要明确讲清楚：

- 分数较低 `不等于反感`
- 只是说明这类表达在你这里 `相对不是第一优先级`
- 例如某人“礼物分低”，不是讨厌收礼物，而是“如果只有礼物、没有时间与表达，他仍可能觉得不够”

### 3.2 五种爱情语言长报告版文案

#### 3.2.1 肯定的言辞 `words_of_affirmation`

**主语言时可用文案**

- 你很看重“爱有没有被说出来”。
- 对你来说，真正打动人的，往往不是一句泛泛的“爱你”，而是那些具体、认真、带着观察的表达。
- 你会记得别人是怎么称赞你、理解你、回应你的，也会对敷衍、沉默、理所当然特别敏感。
- 当关系里长期缺少明确表达，你不一定会立刻发作，但心里会慢慢出现“我到底有没有被珍惜”的空缺感。
- 你最容易被打动的方式，是被认真看见、被具体肯定、被清楚地说出来。

**次语言时可用文案**

- 你并不是只听甜言蜜语的人，但一段关系若完全没有语言上的回应，你会明显感觉少了点什么。
- 当对方愿意把欣赏、感谢、偏爱说出口，你会更容易确认这段关系的温度。

**相对低敏感时可用文案**

- 你并非不喜欢被夸，而是相比“说得好听”，你更在意别的爱的证据。
- 如果只有漂亮话，却没有陪伴、行动或其他你真正在意的表达，你不太会因此被真正打动。

**常见误解**

- 别人可能会误会你“需要很多情绪价值”
- 其实你更在意的是：`对方是否愿意认真表达，而不是敷衍带过`

**伴侣行动建议**

- 多说具体内容，少说模板句
- 比起“你很好”，更有效的是“我喜欢你今天处理这件事的方式”
- 在重要时刻给出明确回应，不要总让你去猜

#### 3.2.2 精心的时刻 `quality_time`

**主语言时可用文案**

- 你最需要的，不是形式上的陪伴，而是“这个时刻里你真的在我这里”。
- 对你来说，两个人是否同处一个空间并不够，关键在于对方有没有把注意力、耐心和真实参与感给到你。
- 被打断、被边聊边看手机、被随口敷衍，都会比别人更容易让你觉得失落。
- 你真正会记住的，往往是那些被认真倾听、被完整陪伴、被优先安排进日程的时刻。
- 当一个人愿意把时间给你、把注意力留给你，你会强烈感受到“我被放在心上”。

**次语言时可用文案**

- 你对时间投入很敏感，哪怕主语言不是它，你依然会因为“对方有没有认真在场”来判断关系温度。
- 当陪伴质量很高时，你对这段关系的安全感会明显上升。

**相对低敏感时可用文案**

- 你不排斥陪伴，只是相比“花多久在一起”，你更看重别的表达方式。
- 如果相处时间普通，但其他你更在意的通道被满足，你依然可能感觉关系是稳定的。

**常见误解**

- 别人容易以为你“黏人”
- 实际上你更在意的是 `相处质量`，不是机械拉长时长

**伴侣行动建议**

- 见面时减少分心，别把陪伴做成背景音
- 给出完整、稳定、可感知的相处时段
- 当你倾诉时，先听完再建议

#### 3.2.3 接受礼物 `receiving_gifts`

**主语言时可用文案**

- 你在意的不是价格，而是那种“对方有把我记在心里”的具体证据。
- 对你来说，一份礼物的价值常常不在贵，而在准：它是否真的贴近你的喜好、回忆、需要与被惦记感。
- 你容易被那些带着观察、纪念和心思的小东西打动，因为它们会把爱从抽象情绪变成能被看见、留住、反复回想的证据。
- 如果一段关系总停留在口头表达，却很少有可触摸、可留存的心意，你可能会觉得少了一层“被认真对待”的质感。

**次语言时可用文案**

- 你会被礼物打动，尤其当它不是完成任务式送出，而是真的和你有关。
- 它像是你关系里的“加深记忆点”，会让你更容易记住被爱的瞬间。

**相对低敏感时可用文案**

- 你不是不喜欢收礼物，而是礼物本身通常不是你判断爱的主要通道。
- 如果只有礼物、没有其他更核心的表达，你未必会因此感到足够满足。

**常见误解**

- 别人容易把你误解成“物质”
- 实际上你更在意的是：`对方有没有在细节上记住你、用心对待你`

**伴侣行动建议**

- 不一定贵，但一定要“像你”
- 把礼物和具体记忆连接起来，比随手买更有感
- 平时的小心意，常常比节日的标准化礼盒更打动你

#### 3.2.4 服务的行动 `acts_of_service`

**主语言时可用文案**

- 你对“爱有没有落到行动上”非常敏感。
- 对你来说，真正让人安心的，往往不是说得多漂亮，而是对方有没有愿意动手、分担、照顾、落实。
- 当你累、乱、忙的时候，有人替你把事情接过去、把细节处理好、把负担分掉，你会比收到很多漂亮话更有被爱的感觉。
- 你会把很多实际行动视作爱的证明：记得、做到、提前安排、主动帮忙、减少你的负担。
- 如果一个人总是“嘴上有你，行动里没你”，你会很快失去耐心。

**次语言时可用文案**

- 你对行动兑现度很看重，虽然不一定排第一，但它会直接影响你对关系可靠性的判断。
- 对你来说，照顾不是浪漫的对立面，反而是很真实的浪漫。

**相对低敏感时可用文案**

- 你当然会感谢别人帮忙，但相比“做了什么”，你可能更先被别的通道打动。
- 单纯的照顾与安排，不一定足以让你感觉情感被真正触达。

**常见误解**

- 别人容易把你看成“务实到不浪漫”
- 其实你只是更相信：`爱如果是真的，通常会体现在愿不愿意做事`

**伴侣行动建议**

- 少一点“你要我做什么就说”，多一点主动察觉
- 比起空头承诺，按时出现、落实细节更能打动你
- 在你最忙最乱时伸手帮一把，效果远胜临时安慰

#### 3.2.5 身体的接触 `physical_touch`

**主语言时可用文案**

- 你对身体距离很敏感，亲密感常常是通过“靠近”被真正确认的。
- 一个拥抱、一只牵住的手、靠在一起的安静时刻、情绪不好时被抱住安抚，这些都会直接影响你对关系温度的感受。
- 对你来说，身体接触不是可有可无的小动作，而是一种非常直接的“我在你身边”的信号。
- 当关系里长期缺少自然靠近、亲密触碰或身体上的回应，你会很容易感觉疏离，即使对方口头上说了很多。

**次语言时可用文案**

- 你对身体靠近有明显好感，它会增强你被在乎、被接住的感受。
- 当接触和你的主语言一起出现时，你会更快进入关系里的安心状态。

**相对低敏感时可用文案**

- 你并不排斥亲密接触，只是它通常不是你判断关系好坏的第一信号。
- 如果别的核心通道没有被满足，再多拥抱也未必能真正解决你的失落感。

**常见误解**

- 别人可能会误会你“过度依赖亲密动作”
- 其实你更在意的是：`通过身体靠近确认彼此没有被推远`

**伴侣行动建议**

- 见面时主动靠近，比等你开口更有感
- 在你情绪不稳时，先给稳定、温柔、尊重边界的接触
- 不要只在热烈时靠近，日常里的自然触碰更有安全感

### 3.3 主语言 × 次语言组合导语

说明：

- 当第一名与第二名差距较小，建议结果页首屏或第二屏补充 `双通道导语`
- 这部分比单独贴一个标签更容易让用户觉得“说到了我”

| 组合 | 导语 |
| --- | --- |
| 言辞 × 时刻 | 你既需要被认真说出来，也需要被认真放在当下。只有表达、没有投入不够；只有陪伴、没有回应也不够。 |
| 言辞 × 礼物 | 你在意的是“被记住”与“被清楚表达”。一句有分量的话，加上一份带记忆点的小心意，会非常打动你。 |
| 言辞 × 行动 | 你很难被空话打动，但也不喜欢沉默式付出。对你最有效的是：说到，也做到。 |
| 言辞 × 接触 | 你需要的是语言上的确认，加上身体上的靠近。被温柔说出口、再被靠近，最容易让你放下防备。 |
| 时刻 × 礼物 | 你既在意“陪你的人是不是真的在场”，也在意“这段关系有没有被具体记住”。时间和心意都要落地。 |
| 时刻 × 行动 | 你重视高质量相处，也重视实际照顾。你要的不是形式浪漫，而是有人认真陪你，也认真帮你。 |
| 时刻 × 接触 | 你需要的是有存在感的陪伴。一起待着、认真聊天、自然靠近，这些都会让你迅速感到安心。 |
| 礼物 × 行动 | 你最容易被“心意”与“落实”一起打动。光有惊喜不够，真正照顾到你会更有分量。 |
| 礼物 × 接触 | 你既会被有纪念感的小心意打动，也会被真实靠近安抚到。你在意关系里有没有“可感知的温度”。 |
| 行动 × 接触 | 你偏爱的爱不是高调表达，而是可靠地出现、照顾、靠近。有人把你接住，这件事本身就很浪漫。 |

### 3.4 结果页模块稿（适合长图与详情页）

#### 3.4.1 结果页目标

- 让用户觉得：`这不是一句套话，而是真的把我在关系里的“有感点”说细了`
- 让伴侣也能看懂：`以后怎么做，会更有效`
- 便于导出长图，适合分享到小红书、聊天窗口、朋友圈

#### 3.4.2 推荐结构

1. `封面模块`
2. `五维分布模块`
3. `主语言深读模块`
4. `次语言补充模块`
5. `最容易失落的关系瞬间`
6. `伴侣行动清单`
7. `低敏感区提醒`
8. `分享尾屏`

#### 3.4.3 模块稿

**模块 1｜封面**

- 主标题：`你的爱情语言主通道是「精心的时刻」`
- 副标题：`你最容易因“被专注陪伴”而感到被爱。`
- 标签：`主语言` / `次语言` / `双主导` / `整体偏均衡`

**模块 2｜五维分布**

- 显示五维雷达图或五维条形图
- 标注每一项的 `分数 / 12` 与简短注释
- 例：`精心的时刻 10/12：你很在意对方是否真的在场`

**模块 3｜主语言深读**

- 标题：`什么最容易让你有“被爱感”`
- 内容：主语言长文案 + 3 条高频触发点
- 示例字段：`loveTriggers[]`

**模块 4｜次语言补充**

- 标题：`除了主通道，你还很在意这一种`
- 内容：次语言为何重要、它与主语言如何叠加

**模块 5｜你最容易失落的瞬间**

- 标题：`关系里，你最容易卡住的点`
- 可按主语言输出不同版本：
  - 言辞主导：`长期没有明确回应、总是让我猜`
  - 时刻主导：`人在我面前，注意力却不在我这里`
  - 礼物主导：`重要时刻总是被忘记、心意感太弱`
  - 行动主导：`嘴上在乎，关键时刻却接不住`
  - 接触主导：`关系里越来越少自然靠近与安抚`

**模块 6｜伴侣行动清单**

- 标题：`如果想让你更有被爱感，可以这样做`
- 每个主语言给 `3~5` 条具体动作建议
- 这部分最适合做成长图里的可收藏内容

**模块 7｜低敏感区提醒**

- 标题：`分数较低，不代表不需要`
- 文案建议：
  - `你的低分维度，不是禁区，只是相较之下没那么容易直接击中你。`
  - `如果关系里只有这一种表达，而缺少你真正高敏感的通道，你仍然可能觉得不够。`

**模块 8｜分享尾屏**

- 短句示例：
  - `原来我不是难哄，我只是被爱的通道不一样。`
  - `不是你没表达，是你表达的方式刚好没打到我。`
  - `我的爱情语言结果，居然真的解释了很多关系小误会。`

### 3.6 文案边界提醒

- 不要把结果写成：`你就是这种人，所以注定怎样`
- 不要把低分维度写成：`你不需要这个`
- 不要把“接受礼物”写成拜金，也不要把“身体接触”写成轻浮
- 更合适的说法是：`在这套题里，你更容易被哪一种表达方式击中`
