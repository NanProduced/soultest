## 1. Landing Page 的真实价值

Landing Page 的价值主要有四个：

1. **承接分享流量**：好友扫码结果长图上的二维码后，不会直接掉进验证码页面
2. **建立品牌感**：让用户感知这是一个品牌平台，而不是散装 H5
3. **促进交叉购买**：用户看完一个测试，可以继续浏览其他题集
4. **承接免费试玩**：为免费测试或活动测试提供统一入口

### 1.1 不应过度承诺的点

在当前 `React SPA + Cloudflare Pages` 架构下，Landing Page 的主要价值是 **品牌与流量承接**，而不是强 SEO。

可以获得的收益：

- 更好的品牌展示
- 更顺滑的分享闭环
- 更高的交叉购买机会

不建议当前阶段把它当作：

- 核心自然搜索增长引擎
- 复杂内容站
- 大型 SEO 落地页体系

如果未来真的要做 SEO，再单独评估静态预渲染或独立增长页方案。

## 2. 页面目标

Landing Page 的页面目标应控制在三件事：

1. 让用户快速理解“灵测 SoulTest 是什么”
2. 让用户看到“有哪些测试值得买/值得试”
3. 让用户完成“去购买 / 去输入验证码 / 先试玩”中的一个动作

---

## 3. 推荐页面结构

### 3.1 V1 简版（建议先做这个）

```text
[1] Hero
[2] 测试展厅
[3] Footer / 店铺二维码
```

### 3.2 V2 完整版（后续再加）

```text
[1] Hero
[2] 测试展厅
[3] 免费体验区
[4] 结果展示墙
[5] 平台特色
[6] FAQ
[7] Footer
```

### 3.3 为什么先做简版

因为 Landing Page 当前不是卖货主链路核心，先把最小承接价值跑通即可：

- 有品牌
- 有测试列表
- 有购买导流

## 4. 关键模块说明

### 4.1 Hero

- 品牌名：`灵测 SoulTest`
- 一句话价值：强调“趣味测试 + 精美结果 + 可分享”
- 主 CTA：`开始探索`
- 次 CTA：`去小红书购买`

推荐文案方向：

- 发现未知的你
- 每一道题，都在靠近真实的你
- 测完不只是一个结果，而是一张值得分享的海报

### 4.2 测试展厅

卡片展示信息建议统一：

- 封面图
- 标题
- 一句话简介
- 标签（热门 / 新上 / 免费）
- 预计时长
- 价格或“已购可进入”提示

点击策略建议：

- 免费测试：直接进入介绍页或答题页
- 付费测试：进入 `/:slug` 介绍页，再进行购买引导或验证码输入

### 4.3 免费体验区（后续）

适合承接：

- 极轻量 range 测试
- 活动推广测试
- 品牌引流测试

免费体验的结果页仍要带回平台入口，形成回流。

### 4.4 结果展示墙（后续）

其核心价值不是“凑热闹”，而是：

- 展示结果页设计水准
- 提供社会证明
- 强化“测试结果本身值得分享”的产品心智

### 4.5 FAQ

推荐 FAQ 统一口径：

- 测试怎么购买？
- 验证码怎么用？
- 结果是否仅供娱乐参考？
- 激活码是否会立即失效？
- 结果图如何保存与分享？

其中验证码说明应更新为：

- **验证码在有效期内可继续进入已授权测试**
- **不采用一次进入就失效的策略**

## 5. 分享二维码策略

### 5.1 默认推荐

结果长图二维码优先指向：

```text
/ 或 /:slug
```

而不是直接指向验证码页。

### 5.2 原因

- 没有码的好友直接进验证码页体验会断裂
- Landing Page 或介绍页更适合承接新用户
- 可以顺带展示更多测试，提升交叉购买概率

### 5.3 推荐优先级

1. **品牌级 Landing Page**：适合承接泛流量
2. **题集介绍页 `/:slug`**：适合承接单一测试的分享流量

后续可根据实际转化数据再做 A/B 测试。

## 6. 技术实现建议

### 6.1 路由建议

```text
/                Landing Page
/:slug           测试介绍页 / 验证码入口页
/admin           管理后台
```

### 6.2 数据来源

Landing Page 只需要极少数公共数据：

- `GET /api/quizzes/public`
- 可选的统计摘要接口 `GET /api/admin/analytics/summary` 的公共裁剪版

### 6.3 动画与性能

- 用 `Framer Motion` 做轻量入场动画即可
- 不引入重型 3D 动画库
- 移动端优先，保持首屏轻量

---

## 7. 开发顺序建议

### P2 / P3 的推荐落地顺序

1. 先做 Hero + 测试展厅 + Footer
2. 接上公开题集卡片数据接口
3. 验证分享流量是否会回流购买
4. 再决定是否补免费体验区、结果墙和 FAQ

---

## 8. 结论

Landing Page 值得做，但应明确它的定位：

- 它是 **品牌与流量承接页**
- 不是当前平台成立的前提条件
- 不宜在项目初始化阶段抢占后台与题集引擎的优先级

因此，建议先保留方案、控制范围，在首批题集和后台能力跑通后，以简版快速落地。
